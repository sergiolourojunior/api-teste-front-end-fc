Documentação

Utilize esta documentação para interagir com as rotas desta API.

Configurações

Todos as rotas partirão da URL http://localhost:3000.

O único método aceito é o GET, utilizado para consultar dados.

O conteúdo de todas as respostas estará formatado em Application/json.

Profissionais

GET /profissionais

Descrição

Esta rota retorna todos os profissionais cadastrados na plataforma.

Requisição

http://localhost:3000/profissionais

Filtro

Para filtrar os resultados, utilize como parâmetro o atributo que você deseja filtrar na URL da requisição. Por exemplo:

http://localhost:3000/profissionais?cidadeId=2

O retorno desta requisição será todos os profissionais que possuem o valor 2 no atributo cidadeId. Caso não existam profissionais cadastrados nesta cidade, será retornado um array de objetos vazio.

Retorno

[
    {
        "id": 1,
        "nome": "Lucca Cláudio Otávio Galvão",
        "cpf": "91176857029",
        "cidadeId": 1,
        "especialidadeId": 1
    },
    {
        "id": 2,
        "nome": "Dr. Bryan Arthur Juan Assunção",
        "cpf": "86821447004",
        "cidadeId": 4,
        "especialidadeId": 1
    },
    // ...
]
Parâmetro Tipo Descrição
id integer Identificador único do registro.
nome string Nome do profissional.
cpf string CPF do profissional.
cidadeId integer Identificador para a rota de cidades.
especialidadeId integer Identificador para a rota de especialidades.

Cidades

GET /cidades

Descrição

Esta rota retorna todos as cidades cadastradas na plataforma.

Requisição

http://localhost:3000/cidades

Filtro

Para filtrar os resultados, utilize como parâmetro o atributo que você deseja filtrar na URL da requisição. Por exemplo:

http://localhost:3000/cidades?estadoId=2

O retorno desta requisição será todas as cidades que possuem o valor 2 no atributo estadoId.

Retorno

[
    {
        "id": 1,
        "nome": "Pelotas",
        "estadoId": 1
    },
    {
        "id": 2,
        "nome": "Santa Maria",
        "estadoId": 1
    },
    {
        "id": 3,
        "nome": "Joinville",
        "estadoId": 2
    },
    // ...
]
Parâmetro Tipo Descrição
id integer Identificador único do registro.
nome string Nome da cidade.
estadoId integer Identificador para a rota de estados.

Estados

GET /estados

Descrição

Esta rota retorna todos os estados cadastrados na plataforma.

Requisição

http://localhost:3000/estados

Filtro

Para filtrar os resultados, utilize como parâmetro o atributo que você deseja filtrar na URL da requisição. Por exemplo:

http://localhost:3000/estados?sigla=RS

O retorno desta requisição será todas as cidades que possuem o valor RS no atributo sigla.

Relacionamento

Você pode fazer relações entre rotas informando na URL com qual tabela você deseja relacionar. Por exemplo:

http://localhost:3000/estados?_embed=cidades

No exemplo acima, o retorno será a lista de estados e para cada estado será listado as cidades respectivas.

Você também pode fazer a relação de apenas um registro. No exemplo abaixo é mostrado como mostrar os dados do estado "Santa Catarina" e suas respectivas cidades.

http://localhost:3000/estados/2/cidades

Retorno

[
    {
        "id": 1,
        "nome": "Rio Grande do Sul",
        "sigla": "RS"
    },
    {
        "id": 2,
        "nome": "Santa Catarina",
        "sigla": "SC"
    },
    // ...
]
Parâmetro Tipo Descrição
id integer Identificador único do registro.
nome string Nome do estado.
sigla string Sigla do estado.
